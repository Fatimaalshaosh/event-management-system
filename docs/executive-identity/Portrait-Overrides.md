# Portrait Overrides

An **override** pins a permanent, human-curated portrait to a specific
`employeeId`. When an override exists the AI provider is **never** called for
that person: the portrait is never generated, regenerated, or replaced.

> See also: [Architecture](./Executive-Identity-Architecture.md) ·
> [Portrait Generation Engine](./Portrait-Generation-Engine.md) ·
> [Maintenance](./Maintenance.md)

## Where it lives

`artifacts/presidential-protocol/src/lib/identity/service.ts`:

```ts
export const PORTRAIT_OVERRIDES: Record<string, string> = {
  // Fatima Darwish — Chief of Protocol (signed-in executive). Permanent official portrait.
  me: "/portraits/fatima-darwish.png",
};
```

The override is checked **first** — before the cache and before any provider —
in both `getPlaceholder()` and `resolve()`. Because every Executive Identity
surface resolves through `portraitService`, a single override applies everywhere
(top nav, sidebar, Contacts, hover card, Mission Engine, Tasks, Approvals,
mentions, timeline, search, people picker, org chart, calendar, notifications).

The batch script also skips override ids (`OVERRIDE_IDS`), so they are never
regenerated.

## Asset location

Override images live in `artifacts/presidential-protocol/public/portraits/` and
are served at `/portraits/<file>` (Vite serves `public/` at the web root).
A single high-resolution square image is sufficient — `ExecutiveAvatar` renders
it with `object-fit: cover` (face-centered) at every size. Optional pre-sized
variants (`-48`, `-64`, `-96`, `-256`) may be added for network optimization but
are not required.

## How to add an override

1. Save the curated image to `public/portraits/<name>.png` (high-res, square,
   face centered).
2. Add one line to `PORTRAIT_OVERRIDES`, keyed by the person's `employeeId`.
3. Bump `PORTRAIT_VERSION` and restart the frontend so it appears immediately.

### Example: Fatima Darwish (the signed-in executive, employeeId `"me"`)

```ts
export const PORTRAIT_OVERRIDES: Record<string, string> = {
  me: "/portraits/fatima-darwish.png",
};
```

### Example: a future executive (numeric employeeId)

```ts
export const PORTRAIT_OVERRIDES: Record<string, string> = {
  me: "/portraits/fatima-darwish.png",
  "42": "/portraits/director-general.png", // employeeId 42 — official photo
};
```

### Example: a minister with an official portrait

```ts
  "118": "/portraits/minister-of-state.png", // H.E. the Minister — official portrait
```

### Example: a chairman

```ts
  "22": "/portraits/vice-chairman.png", // pin the Vice Chairman to an official photo
```

## Guarantees

For any id present in `PORTRAIT_OVERRIDES`:

- the OpenAI provider is never called,
- the portrait is never regenerated by any batch,
- the image is never replaced by the cache or a provider,
- the exact curated asset is always used, everywhere.
