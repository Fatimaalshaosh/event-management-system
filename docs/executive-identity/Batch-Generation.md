# Batch Generation

Portraits are generated by `scripts/src/generate-portraits-full.ts`
(`pnpm --filter @workspace/scripts run portraits:full`). The script is
**safe by design**: it runs exactly **one pass per invocation** and never
generates all batches automatically.

> See also: [Architecture](./Executive-Identity-Architecture.md) ·
> [Portrait Generation Engine](./Portrait-Generation-Engine.md) ·
> [Maintenance](./Maintenance.md)

## How batches work

- People (`internal`, `external`, `vip`) are collected from the API; **organizations
  are skipped**, and ids in `OVERRIDE_IDS` (e.g. Fatima `"me"`) are skipped.
- The list is processed in fixed batches of **15** (`BATCH = 15`).
- Each generated file is `sha1(employeeId)` so it is owned by the employee.
- The script verifies each file exists after generation and retries a failure
  once; it never regenerates a portrait that already exists (the route returns
  it as cached).

The 87-person directory was generated in 6 batches (`BATCH_INDEX` 0–5; the last
batch is 12 people).

## Prerequisites

The api-server must be running with the OpenAI key in its environment, started
from the **repository root** (so `.portraits/` resolves at the root):

```bash
# from the repo root
set -a; . ./.env; set +a
PORT=8080 node ./artifacts/api-server/dist/index.mjs
```

Point the script at it with `PORTRAIT_API` (defaults to `http://localhost:8080`).

## `BATCH_INDEX` — run one batch

`BATCH_INDEX=N` generates people `[N*15 .. N*15+15)` and then **pauses**,
printing `re-run with BATCH_INDEX=N+1 to continue`. There is no auto-continue —
each batch is a manual, reviewable step.

```bash
PORTRAIT_API=http://localhost:8080 BATCH_INDEX=0 \
  pnpm --filter @workspace/scripts run portraits:full
# review, then:
PORTRAIT_API=http://localhost:8080 BATCH_INDEX=1 \
  pnpm --filter @workspace/scripts run portraits:full
```

## `ONLY_IDS` — regenerate specific employees

`ONLY_IDS=<comma-separated employeeIds>` regenerates only those people and
ignores `BATCH_INDEX`. Use it for targeted fixes.

```bash
PORTRAIT_API=http://localhost:8080 ONLY_IDS=73,111,72 \
  pnpm --filter @workspace/scripts run portraits:full
```

## How to regenerate safely

Because a portrait is **cached forever once the file exists** (and the filename
is stable `sha1(employeeId)`), regenerating requires deleting the old file(s)
first, then running the script:

```bash
# 1. delete the target files (example: a single batch's 15 people)
node -e '
const cp=require("crypto"),fs=require("fs");
const PEOPLE=new Set(["internal","external","vip"]), OV=new Set(["me"]);
const f=id=>cp.createHash("sha1").update(String(id)).digest("hex").slice(0,32)+".png";
fetch("http://localhost:8080/api/contacts").then(r=>r.json()).then(cs=>{
  const people=cs.filter(c=>PEOPLE.has(c.type)&&!OV.has(String(c.id)));
  for(const c of people.slice(0,15)){ const p=".portraits/"+f(c.id); if(fs.existsSync(p)) fs.unlinkSync(p); }
});
'

# 2. regenerate those people
PORTRAIT_API=http://localhost:8080 BATCH_INDEX=0 \
  pnpm --filter @workspace/scripts run portraits:full

# 3. bump the cache version so browsers fetch the new images (see below)
```

Safety rules:
- **Never delete or regenerate** files for ids you do not intend to change.
- **Never touch** override ids (Fatima `"me"`); they are skipped automatically.
- **Validate after every batch:** employeeId linkage, no missing files, no
  duplicate faces (byte-level), correct nationality/gender/attire/age, Arabic
  RTL + English LTR, no broken images.

## How cache versioning works

The filename never changes, so a regenerated image collides with the browser's
immutable cache. The fix is `PORTRAIT_VERSION` in
`artifacts/presidential-protocol/src/lib/identity/service.ts`:

- it is appended to every portrait URL as `?v=<PORTRAIT_VERSION>`, and
- it is part of the localStorage cache namespace.

**After regenerating, bump `PORTRAIT_VERSION`** (e.g. `2026-06-29.4` →
`2026-06-29.5`) and restart the frontend dev server. Every browser then fetches
the new images automatically and the frontend cache re-resolves — no manual
cache clearing. See [Maintenance](./Maintenance.md#bumping-portrait_version).

## Validation snippet (full directory)

```bash
node -e '
const cp=require("crypto"),fs=require("fs");
const PEOPLE=new Set(["internal","external","vip"]), OV=new Set(["me"]);
const f=id=>cp.createHash("sha1").update(String(id)).digest("hex").slice(0,32)+".png";
const sha256=p=>cp.createHash("sha256").update(fs.readFileSync(p)).digest("hex");
fetch("http://localhost:8080/api/contacts").then(r=>r.json()).then(cs=>{
  const people=cs.filter(c=>PEOPLE.has(c.type)&&!OV.has(String(c.id)));
  let present=0; const byContent={}; const missing=[];
  for(const c of people){ const p=".portraits/"+f(c.id);
    if(fs.existsSync(p)){present++; const h=sha256(p); (byContent[h]=byContent[h]||[]).push(c.id);}
    else missing.push(c.id); }
  const dups=Object.values(byContent).filter(g=>g.length>1);
  console.log("linked:",present,"/",people.length,"| missing:",missing.length,"| dup-faces:",dups.length);
});
'
```
