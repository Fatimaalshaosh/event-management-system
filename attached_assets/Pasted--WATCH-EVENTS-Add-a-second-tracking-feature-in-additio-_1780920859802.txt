WATCH EVENTS
━━━━━━━━━━━━━━━━━━━━

Add a second tracking feature in addition to Pinning:

👀 Watch Event

This is different from Pinning.

Pinned Events:

* Used for prioritizing important events.
* Manually organized by the user.

Watched Events:

* Used for monitoring event changes and operational updates.

Each event card should include:

⭐ Pin / Unpin
👀 Watch / Unwatch

WATCHED EVENTS BEHAVIOR
━━━━━━━━━━━━━━━━━━━━

When a user watches an event:

The system automatically monitors:

* Readiness changes
* Task completion status
* New approvals
* Approval delays
* Risk level changes
* Delegation updates
* Schedule changes
* Venue changes
* Fleet assignment changes
* Hospitality updates
* Security updates
* VIP attendance changes

SMART NOTIFICATIONS
━━━━━━━━━━━━━━━━━━━━

Generate intelligent notifications when:

* Readiness drops below target
* Critical task becomes overdue
* New approval is received
* Approval is rejected
* Event status changes
* New risk is detected
* Event approaches within 72 hours
* VIP information is modified
* Transportation readiness becomes critical

Examples:

⚠️ Readiness for Gulf Cooperation Summit dropped from 92% to 84%.

⚠️ Catering approval is still pending and may impact event readiness.

✅ Security coordination has been approved.

⚠️ Fleet allocation conflict detected.
